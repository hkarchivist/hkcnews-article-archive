# 【831太子站】救護指揮官36分鐘無向消防通訊中心匯報　翻查手稿紀錄亦無法解釋傷者人數、傷勢變化

8.31太子站傷者情況持續引起輿論猜疑，消防處昨天第5度就當晚情況見記者釋疑。眾新聞綜合立法會議員楊岳橋公開的8月31日晚至9月1日凌晨消防處個案紀錄（incident log），以及消防處早前的解說，歸納出以下3個關鍵時間點：


|   
 **9.1**   
 **凌晨** | **詳情** |
| --- | --- |
| 00:15 | 消防處的救護指揮官首次向通訊中心匯報，現場有10名傷者，包括6名為紅色（嚴重）傷者、2名黃色（普通）及2名綠色（輕傷） |
| 00:20 | 醫管局為10名傷者作分流指示，計劃送往廣華醫院、伊利沙伯醫院、明愛醫院及瑪嘉烈醫院。 |
| 01:02 | 消防處的救護指揮官再向通訊中心匯報，修改傷者數字為7人，包括3名為紅色（嚴重）傷者、2名黃色（普通）及2名綠色（輕傷） |


在9月1日凌晨 00:20至01:02的42分鐘之間，現場人員僅於00:26曾向通訊中心匯報，指消防處的高級救護主任進入太子站，但期間沒有其他關乎傷者的匯報紀錄。有現職消防處救護員對眾新聞表示情況不尋常。消防處高級助理救護總長（港島及九龍區域）羅信堂回應指， 前線救護指揮官當晚有紀錄傷者變化，惟只有很「粗略」、「不完整」的手寫紀錄，未能作準，消防處至今未能確認現場傷者人數及傷勢變化的確實情況。

### 傷者數字「10變7」之間無官方紀錄   
 羅信堂：有不完整手寫紀錄傷者變化，但不能作準

有不願具名的現職救防處救護員分析指，救護指揮官到達救援現場後，對傷者的初步檢查未必非常詳細，故首個匯報紀錄的傷者人數及傷勢往往未能作準，其後的確有機會調整，甚至將傷勢降級。惟他指出，救護指揮官會向通訊中心密切匯報傷者的變化，如果就人數、傷勢有改動，救護指揮官一般會逐個傷者匯報、交代更改傷勢的原因，而消防處就8.31太子站的個案紀錄中，超過半小時未有匯報，期間沒有交代傷者人數及傷勢變化，做法並不尋常。

消防處高級助理救護總長（港島及九龍區域）羅信堂在周四（19日）的記者會上回應指，一般而言，前線人員會向通訊中心匯報傷者數目或情況的變化，但8.31太子站事故較為特殊，因為現場只有一位救護指揮官，他當時忙於紀錄資料，該些是很基本、草稿形式的資料，所以初步匯報時，他只有概括表示傷者數字為10至15；當他進一步檢查後，再匯報傷者數字為10人，而當時仍未紀錄傷者的年齡；及後支援同事到場，資料更為確切，救護指揮官再向通訊中心匯報為傷者為7人。

羅信堂形容，前線救護指揮官當晚有紀錄傷者變化，但在一個「兵荒馬亂」的情況下，只有很「粗略」、「初步」、「不完整」的手寫紀錄，救護指揮官事後亦未必憶述到當時為何寫下某些字詞。消防處事後曾檢閱相關手稿紀錄，認為僅可作參考，最終傷者數目及情況仍以救護車送院時為準。

消防處副消防總長（總部）陳慶勇補充指，相關同事當時快速以手稿紀錄，事後被詢問時亦未能確切說明傷者變化。基於當事人未能確認資料的準確性，消防處不能以之為官方紀錄，亦不能公開，否則可能會誤導公眾人士。

### 救護指揮官36分鐘「0通報」   
 陳慶勇：正常合理情況

陳慶勇確認，在9月1日凌晨 00:26至01:02之間的36分鐘，現場救護指揮官與通訊中心並無聯絡，故沒有錄音紀錄可以公開。雖然個案紀錄在此期間有紀錄，但該些訊息都與太子站月台的事故無關。陳慶勇解釋，00:26時，增援的救護人員剛進入太子站，支援單獨在站內工作的救護指揮。在該段時間內，救護指揮需要告訴增援救護人員傷者位置、初步估算傷勢，並要治理傷者，及集中起來點算。完成上述工作後，救護指揮向通訊中心匯報確認有7名傷者，當時已是01:02。他重申，在該36分鐘內，救護指揮要應付大量工作，相當忙碌。

被問到救護指揮長達36分鐘沒有向通訊中心作任何匯報是否正常，陳慶勇指，前線人員一般相隔30至45分鐘會向通訊中心通報訊息，而通訊中心亦會在30至45分鐘左右問現場人員有沒有新的訊息。在8.31太子站個案中，通訊中心尚未「追」現場同事， 現場同事已向通訊中心發送訊息，他認為相關救護指揮在36分鐘後向通訊中心匯報屬正常、合理的情況。
